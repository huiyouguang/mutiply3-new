## 1 销售循环


### 1.1 营业收入（一般）实质性程序
1. 目标：发生、完整性、准确性、截止、分类、列报
2. 获取营业收入明细表
3. 实施实质性分析程序：差异超过可接受差异额，需要对差异额的全额进行调查
4. 检查主营业务收入确认方法是否符合企业会计准则的规定
5. 检查交易价格
6. 逆查（查存在）
7. 顺查（查完整）
8. 函证（结合应收账款）
9. ! 主营业务收入的截止测试
10. 选取资产负债表日前后若干天的出库单，与应收账款和收入明细账进行核对；同时，从应收账款和收入明细账选取在资产负债表日前后若干天的凭证，与出库单核对
	1. 复核资产负债表日前后销售和发货水平，确定业务活动水平是否异常
	2. 取得资产负债表日后所有的销售退回记录，检查是否存在提前确认收入的情况
	3. 结合对资产负债表日应收账款/合同资产的函证程序，检查有无未取得对方认可的销售
11. 检查销售退回
		1. 业务流程
			1. ! **双向核对**（单↔账）
			2. ! → **异常分析**（看货物水平，看前后波动）
			3. ! → **退回排查**（查日后退货，看是否提前确认收入）
			4. ! → **函证补漏**（找客户佐证，找认可销售）
12. 检查可变对价的会计处理
13. 检查主营业务收入在财务报表中的列报和披露是否符合企业会计准则的规定
14. 脉络：
	1. → 拿明细表
	2. → 做分析程序找异常
	3. → 查确认方法 + 交易价格是否合规
	4. → 用逆查 + 顺查验证存在和完整
	5. → 发函证给客户
	6. → 做截止测试防提前 / 延后确认
	7. → 处理销售退回 + 可变对价
	8. → 检查列报披露
	9. → 必要时做延伸检查（走访→查人→查货→查钱）
		


### 1.2 营业收入的“延伸检查”程序（走访→查人→查货→查钱）
对相关供应商、客户进行实地走访
利用企业信息查询工具，查询主要供应商和客户的股东至其最终控制人
检查经销商的最终销售实现情况
获取并检查相关关联方的银行账户资金流水



### 1.3 🧠 应收账款实质性程序 串联记忆法
#### 1.3.1 先抓核心审计目标
**口诀**：**「存完权准分列」**
- 存：存在
- 完：完整性
- 权：权利和义务
- 准：准确性、计价和分摊
- 分：分类
- 列：列报



#### 1.3.2 完整流程串联记忆
我们可以用 **「拿表→分析→函证→细节测试→坏账→列报」** 的逻辑链条来记忆：
1.  **拿表**：取得应收账款明细表
    - 这是所有程序的起点，用于掌握应收账款的整体情况。
2.  **分析**：分析与应收账款相关的财务指标
    - 比如应收账款周转率、账龄分析等，通过趋势和比率发现异常波动。
3.  **函证**：对应收账款实施函证程序
    - 这是验证应收账款**存在性**和**权利义务**的核心外部程序，直接向客户发函确认余额。
4.  **细节测试**：对应收账款余额实施函证以外的**细节测试**
    - 针对未回函、函证无效或特殊项目，通过检查合同、发票、发货单等凭证补充验证。
5.  **坏账**：检查坏账的冲销和转回
    - 关注坏账准备计提是否充分、冲销和转回是否符合准则，确保计价准确。
6.  **列报**：确认应收账款的列报是否恰当

    - 检查报表中是否按准则要求分类披露（如单项金额重大、账龄分布等），保证信息透明。



# 采购循环

### 0.1 🧠 采购与付款循环审计 串联记忆法
#### 0.1.1 整体框架：风险评估 → 风险应对
**核心逻辑**：先识别风险，再通过控制测试+实质性程序应对风险。
- **风险评估**：了解主要业务活动 + 内部控制（职责分离）
- **重大错报风险**：识别交易/余额风险 → 评价固有风险和控制风险
- **风险应对**：控制测试 + 应付账款实质性程序



#### 0.1.2 应付账款实质性程序 核心脉络
##### 0.1.2.1 （1）基础目标
**口诀**：**「存完权准分列」**
- 存：存在
- 完：完整性
- 权：权利和义务
- 准：准确性、计价和分摊
- 分：分类
- 列：列报

##### 0.1.2.2 获取或编制应付账款明细表
##### 0.1.2.3 （2）核心程序：函证应付账款
- **方向**：从非财务部门（如采购部门）获取供应商清单选取样本
- **对象**：不能只挑大额/大供应商，还要关注**小余额或零余额供应商**
- **不符处理**：调查差异 → 检查支持性文件
- **未回函替代**：检查付款单据、采购单据

##### 0.1.2.4 （3）重点程序：查找未入账应付账款（完整性目标）
这是本循环的**核心考点**，可按「查本期→查日后→查存货」串联：
 1. **本期变动检查**：对本期增减变动的应付账款，检查支持性文件，确认会计处理
 2. ! **日后凭证检查**：查资产负债表日后应付账款贷方发生额凭证，获取对账单和差异调节表
 3. @ **日后付款检查**：查资产负债表日后付款项目，检查银行对账单和凭证，查找未及时入账负债
 4. $ **存货入库关联**：结合存货监盘，检查资产负债日前后存货入库资料（验收报告/入库单）
 5. & **费用截止检查**：追踪已选应付账款、货到票未到暂估入账、预提费用明细表，关注费用计入期间

##### 0.1.2.5 （4）其他程序
- 获取/编制应付账款明细表
- 除折旧/摊销、人工费用外的一般费用实质性程序
- 确认应付账款列报是否恰当



#### 0.1.3 简化记忆口诀
**「评风险→做函证→查未入账→列报」**
- 评风险：风险评估+识别重大错报风险
- 做函证：按方向选样本，处理不符与未回函
- 查未入账：本期变动、日后凭证、付款、存货入库、费用截止多维度排查
- 列报：确认报表披露合规






### 0.2 🧠 生产与存货循环审计（存货监盘核心）串联记忆法
#### 0.2.1 整体框架：风险评估 → 风险应对
**核心逻辑**：先识别重大错报风险，再通过控制测试+存货监盘等实质性程序应对。
- **风险评估**：了解主要业务活动 → 评估重大错报风险
- **重大错报风险**：生产与存货循环存在的重大错报风险
- **风险应对**：控制测试 + 存货监盘等实质性程序


- [ ]存货监盘
#### 0.2.2 存货监盘核心框架   
##### 0.2.2.1 （1）基本要求与责任
- **要求**：存货对财务报表重要时，**应当在存货盘点现场实施监盘**（除非不可行）
- **责任划分**：
  - CPA：负责**监盘**
  - 管理层：负责**盘点**
  - 结论：存货监盘**不能替代**管理层定期盘点存货的责任
- **运用环节**：可作为**控制测试**或**实质性程序**；若少数项目构成存货主要部分，可能选择将监盘用作实质性程序

##### 0.2.2.2 （2）监盘目标（认定对应）

| 认定 | 含义 | 典型举例 |
| **存在** | 主要目标，验证存货真实存在 | 期末监盘发现货到单未到的存货未暂估入账 |
| **完整性** | 验证所有存货均已记录 | 同上 |
| **准确性、计价和分摊** | 验证存货金额准确 | 存货发霉、变质导致减值 |
| **权利和义务** | 验证存货归被审计单位所有 | 存货被法院查封 |


💡 记忆：**存在是主要，完整、计价、权利是部分**



#### 0.2.3 制定存货监盘计划的考虑事项
1.  **与存货相关的重大错报风险**
2.  **与存货相关的内部控制性质**：需了解被审计单位存货相关内控
3.  **盘点时间安排**：
    - 若盘点在资产负债表日以外日期进行，需实施其他程序确定盘点日至资产负债表日的存货变动记录恰当
4.  **被审计单位是否采用永续盘存制**
5.  **存货存放地点**：
    - 要求提供完整存货存放地点清单，考虑完整性
    - 包含：仓库、期末存量为零的仓库、租赁的仓库、第三方保管存货的仓库
6.  **是否需要专家协助**

风险控制、时间地点、专家制度

### 0.3 🧠 存货监盘全流程 + 舞弊应对 串联记忆法
#### 0.3.1 一、存货监盘执行流程（第一张图核心）
##### 0.3.1.1 监盘前工作（观察盘点现场）
- **应纳入盘点范围**：存货已恰当整理排列，并附有盘点标识
- **未纳入范围的处理**：
  - 对所有权不属于本单位的存货，取得规格、数量等资料，确认是否单独存放、标明且未纳入盘点
  - 即使被审计单位声明不存在受托代存存货，也需关注是否存在不属于本单位的存货迹象

##### 0.3.1.2 监盘核心程序（口诀：**评观检执**）
- **评**：评价管理层记录和控制存货盘点结果的指令和程序
- **观**：观察管理层盘点程序的执行情况
- **检**：检查存货实物
- **执**：执行**双向抽盘**（盘点记录→实物，实物→盘点记录）
  - 避免让被审计单位事先知晓抽盘项目
  - **发现差异的措施**：
    1. 查明原因，提请被审计单位改正
    2. 考虑错误潜在范围和重大程度，必要时扩大检查范围
    3. 要求重新盘点（范围可限于特定领域或小组）

##### 0.3.1.3 监盘结束工作
- 再次观察盘点现场
- 取得并检查已填用、作废及未使用的盘点表表单，确认是否连续编号



#### 0.3.2 二、特殊情况处理
1.  **监盘不可行**
    - **合理原因**：存货性质/存放地点导致（如安全威胁）
      - 替代程序可行：实施替代审计程序
      - 替代程序不可行：发表非无保留意见
    - **不合理原因**：审计困难、时间或成本，**不能作为**不实施监盘的理由
2.  **不可预见情况无法现场盘点**：另择日期监盘，并对间隔期内交易实施审计程序
3.  **由第三方保管/控制的存货**
    - 向第三方函证存货数量和状况
    - 实施/安排其他注册会计师对第三方存货监盘（如可行）
    - 获取其他注册会计师针对存货盘点和保管内控的报告
    - 检查仓储单等相关文件记录
    - 若存货被抵押，要求其他机构/人员确认
    - 评估第三方保管存货商业理由的合理性，并执行对应审计程序



#### 0.3.3 三、针对存货舞弊风险的应对措施（第二张图核心）
##### 0.3.3.1 虚构存货的应对
- 设计并执行存货监盘程序
- 关注金额大、占比高、库龄长、周转率低的存货，分析合理性
- 执行分析性程序，检查存货结构波动与收入结构变动的匹配性，评价产成品存货与收入、成本的联动关系
- 对异地/第三方保管的存货，严格执行函证或异地监盘

##### 0.3.3.2 账外存货的应对
- 在其他资产审计中，关注是否存在转移资产形成账外存货的情况
- 关注存货盘亏、报废的内控程序，检查是否存在异常大额盘亏/报废
- 存货监盘中，重点关注存货的**所有权**与**完整性**
- 警惕通过多结转成本、多报耗用数量、少报产成品入库等方式形成账外存货



#### 0.3.4 四、简化记忆口诀
**「监盘前→评观检执→结束→特殊处理→舞弊应对」**
- 监盘前：查范围、看标识、问归属
- 核心程序：评观检执，双向抽盘
- 结束：再观察、查表单、保编号
- 特殊情况：不可行找替代，不可预见改日期，第三方靠函证+监盘
- 舞弊风险：虚构查监盘+分析，账外查转移+盘亏+监盘+成本结转





### 0.4 🧠 存货舞弊风险应对 串联记忆法
这张图核心是**存货计价测试**下，针对**虚构存货**和**账外存货**两类舞弊风险的应对措施，我帮你按逻辑串联记忆：



#### 0.4.1 一、两类舞弊风险的核心区别




| 舞弊类型       | 本质                     | 审计目标                                   |
|----------------|--------------------------|--------------------------------------------|
| 虚构存货       | 无中生有，虚增资产       | 验证存货**存在性**、**准确性**             |
| 账外存货       | 有货不入账，隐瞒资产     | 验证存货**完整性**、**权利和义务**         |


#### 0.4.2 二、虚构存货的应对措施（4步）
**口诀**：**监盘→分析→匹配→函证/异地盘**
1.  **监盘打底**：根据存货特点、盘点制度和内控，设计并执行**存货监盘程序**，直接验证实物存在
2.  **异常筛查**：关注金额大、占比高、库龄长、周转率低于同行的存货，分析其合理性
3.  **联动分析**：执行分析性程序，检查存货结构波动与收入结构变动的匹配性，评价产成品存货与收入、成本的联动关系
4.  **异地/第三方验证**：对异地存放或第三方保管的存货，严格执行**函证**或**异地监盘**，防止虚构



#### 0.4.3 三、账外存货的应对措施（4步）
**口诀**：**查转移→查盘亏→查权属→查成本**
1.  **资产联动**：在其他资产审计中，关注是否存在**转移资产**形成账外存货的情况
2.  **内控关注**：关注存货盘亏、报废的内控程序，检查是否存在**异常大额盘亏/报废**（可能是将账内存货转至账外）
3.  **监盘重点**：存货监盘中，重点关注存货的**所有权**与**完整性**，防止漏记
4.  **成本核查**：警惕通过**多结转成本、多报耗用数量、少报产成品入库**等方式，将存货藏于账外



#### 0.4.4 四、整体串联记忆流
> **虚构存货**：先监盘看实物 → 再查异常存货 → 用收入/成本联动分析 → 对异地/第三方存货函证监盘
> **账外存货**：查资产转移痕迹 → 查盘亏报废内控 → 监盘时盯权属和完整性 → 查成本结转是否异常



#### 0.4.5 五、简化记忆口诀
**「虚存监析配函证，账外转亏权属本」**
- 虚存：虚构存货 → 监盘、分析、匹配、函证/异地盘
- 账外：账外存货 → 转移资产、盘亏报废、权属完整性、成本结转




# 货币资金
### 0.1 🧠 货币资金审计 串联记忆法
#### 0.1.1 一、银行存款实质性程序（第一张图核心）
##### 0.1.1.1 基础流程：从余额到发生额
- **获取余额明细表**：
  - 对银行账户完整性存疑时，亲自到人行/基本户开户行打印《已开立银行结算账户清单》，或结合其他细节测试
- **实质性分析程序**：分析货币资金的合理性
- **检查银行存款账户发生额**：
  1. 获取加盖印章的全部银行对账单
  2. 对银行对账单与银行存款日记账进行**双向核对**
  3. 浏览资产负债表日前后的对账单和日记账，关注大额/异常资金变动、红字冲销等，必要时追加程序

##### 0.1.1.2 核心程序：对账单与余额调节表
- **取得并检查银行对账单**（真实性保障）：
  - ! 亲自到银行获取对账单，或观察被审计单位网银导出过程，核对界面真实性
  - 将对账单与银行询证函回函核对，确认一致
  - 将对账单余额与日记账余额核对，存在差异时获取余额调节表
- **取得并检查银行存款余额调节表**：
  1. ! 检查调节表中加计数是否正确
  2. 检查调节事项：
     - ! 企付（收）银未付（收）：检查原始凭证+期后对账单
     - ! 银收（付）企未收（付）：确定是否存在截止错报
  3. 关注长期未达账项，排查挪用资金
  4. ! 特别关注银付企未付、企付银未付中支付异常的领款事项
- **函证银行存款余额**：（第3章核心程序）

##### 0.1.1.3 定期存款 & 其他货币资金
- **定期存款**：
  - ! 比例偏高时，向管理层询问商业理由并评估合理性
  - ! 未质押：检查开户证实书原件；已质押：检查存单复印件+质押合同
  - 函证定期存款信息，分析利息收入合理性
  - ! 检查期后到期/支取的兑付凭证、对账单或网银记录
- **其他货币资金**：
  - 第三方平台资金：了解用途，获取协议，获取平台明细并与账面核对，大额交易追加检查
  - ! 保证金存款、存出投资款：按对应业务实施审计程序



#### 0.1.2 二、货币资金舞弊风险应对（第二张图核心）
##### 0.1.2.1 虚构货币资金
- ! 严格实施银行函证程序，控制函证全过程，调查不符事项
- ! 关注货币资金真实性、大额余额及定期存单合理性
- @ 检查银行账户数量/分布是否匹配经营需求，验证账户完整性和对账单真实性
- $ 分析利息收入与财务费用合理性，关注“存贷双高”现象
- % 关注大额境外资金、无业务支持的大额资金/汇票往来

##### 0.1.2.2 大股东侵占货币资金
- ! 识别银行对账单中与实际控制人、高管的大额资金往来，关注是否基于真实交易
- ! 分析交易信息，识别疑似关联方，检查大额资金/汇票往来的真实性
- ! 关注期后重要账户划转、资金受限情况
- $ 通过公开信息了解实控人/股东财务状况，检查高比例股权质押情况
- $ 结合行业特征，评价现金交易合理性，检查内控是否健全

##### 0.1.2.3 虚构现金交易
- ! 计算月现金收/付款占比，关注与业务性质匹配性，识别异常波动
- ! 了解现金交易对方情况，关注结算合理性和交易真实性
- ! 检查大额现金收支，追踪来源去向，核对原始单据与合同/订单/入库单
- ! 检查交易对象外部证据，验证交易真实性
- ! 排查洗钱等违法违规行为



#### 0.1.3 三、简化记忆口诀
**「银存流程：明→析→发→对→调→函，定期其他补全」**
**「舞弊应对：虚存函析查，侵占联方查，现金交易追」**



#### 0.1.4 四、场景化联想
把整个流程想象成**「查企业钱袋子的完整动作」**：
> 先看有多少钱（余额）→ 看钱怎么进出（发生额）→ 对账找差异（对账单+调节表）→ 找银行确认（函证）→ 特殊资金单独查（定期/第三方）→ 最后防造假：虚构资金、大股东挪用、假现金交易
